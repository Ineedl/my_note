# 常用命令

## 行锁竞争情况查询

mysql版本>5.5

```mysql
show status like 'innodb_row_lock%';
```

```
+-------------------------------+--------+
| Variable_name                 | Value  |
+-------------------------------+--------+
| Innodb_row_lock_current_waits| 2      |
| Innodb_row_lock_time         | 10342  |
| Innodb_row_lock_time_avg     | 21     |
| Innodb_row_lock_time_max     | 2249   |
| Innodb_row_lock_waits        | 482    |
+-------------------------------+--------+
```

| 变量名                          | 含义                                       | 如何解读                                             |
| ------------------------------- | ------------------------------------------ | ---------------------------------------------------- |
| `Innodb_row_lock_current_waits` | 当前正在等待行锁的数量                     | 如果不为 0，说明有事务在等待获取行锁（可能存在阻塞） |
| `Innodb_row_lock_time`          | 自服务器启动以来，行锁等待的总时间（毫秒） | 越大说明历史上等待多，适合做趋势监控                 |
| `Innodb_row_lock_time_avg`      | 每次行锁等待的平均时间（毫秒）             | 平均响应时间指标                                     |
| `Innodb_row_lock_time_max`      | 一次等待行锁的最大时间（毫秒）             | 是否存在极端等待                                     |
| `Innodb_row_lock_waits`         | 自服务器启动以来行锁等待的次数             | 说明系统锁冲突发生频率                               |

## 查询当前 InnoDB 存储引擎中**正在活跃的事务信息**。

mysql版本>5.5

```
select * from information_schema.innodb_trx;
```

### 返回的主要字段说明

| 字段名                  | 含义                                                    |
| ----------------------- | ------------------------------------------------------- |
| `trx_id`                | 事务唯一 ID（内部标识）                                 |
| `trx_state`             | 事务状态（RUNNING、LOCK WAIT、COMMITTED、ROLLING BACK） |
| `trx_started`           | 事务启动时间                                            |
| `trx_requested_lock_id` | 正在请求的锁 ID（如果有等待）                           |
| `trx_wait_started`      | 等待锁的开始时间                                        |
| `trx_weight`            | 事务持有的锁数量和操作复杂度权重                        |
| `trx_mysql_thread_id`   | 关联的 MySQL 线程 ID                                    |
| `trx_query`             | 当前执行的 SQL 语句（如果能获取到）                     |
| `trx_tables_in_use`     | 当前事务涉及的表数量                                    |
| `trx_rows_locked`       | 当前事务锁定的行数                                      |
| `trx_rows_modified`     | 当前事务修改的行数                                      |

## 当前 InnoDB 存储引擎中发生的 **锁等待情况**。

mysql版本>8.0

```mysql
select * from sys.innodb_lock_waits;
```

### 主要返回字段

| 字段名               | 说明                |
| -------------------- | ------------------- |
| `waiting_trx_id`     | 正在等待锁的事务 ID |
| `waiting_thread_id`  | 等待锁的线程 ID     |
| `waiting_query`      | 等待锁的 SQL 语句   |
| `blocking_trx_id`    | 持有锁阻塞的事务 ID |
| `blocking_thread_id` | 阻塞的线程 ID       |
| `blocking_query`     | 阻塞锁的 SQL 语句   |
| `wait_started`       | 等待开始时间        |
| `lock_mode`          | 锁模式，如 `X`、`S` |

## MySQL 服务器内部 **所有线程的信息**

```mysql
select * from performance_schema.threads;
```

### 关键字段说明

| 字段名                | 含义                                               |
| --------------------- | -------------------------------------------------- |
| `THREAD_ID`           | 内部线程唯一标识                                   |
| `PROCESSLIST_ID`      | 对应于 `SHOW PROCESSLIST` 的线程ID（客户端连接ID） |
| `NAME`                | 线程名称                                           |
| `TYPE`                | 线程类型（如 `FOREGROUND`、`BACKGROUND`）          |
| `PROCESSLIST_USER`    | 线程所属的 MySQL 用户名                            |
| `PROCESSLIST_HOST`    | 连接客户端的主机地址                               |
| `PROCESSLIST_COMMAND` | 当前执行的命令类型（Query、Sleep等）               |
| `PROCESSLIST_STATE`   | 当前线程状态描述                                   |
| `PROCESSLIST_INFO`    | 当前执行的 SQL 语句（如果有）                      |
| `THREAD_OS_ID`        | 线程对应的操作系统线程 ID                          |

## **当前会话最近执行的 SQL 语句的历史事件**

```mysql
select * from performance_schema.events_statements_history;
```

### 主要字段说明

| 字段名          | 含义                       |
| --------------- | -------------------------- |
| `THREAD_ID`     | 执行语句的线程 ID          |
| `EVENT_ID`      | 事件唯一标识               |
| `SQL_TEXT`      | 执行的 SQL 语句文本        |
| `TIMER_START`   | 事件开始时间（计时器值）   |
| `TIMER_END`     | 事件结束时间（计时器值）   |
| `TIMER_WAIT`    | 事件持续时间（纳秒）       |
| `LOCK_TIME`     | 锁等待时间                 |
| `ERROR_NUMBER`  | 执行时产生的错误代码       |
| `ROWS_SENT`     | 返回给客户端的行数         |
| `ROWS_EXAMINED` | 扫描的行数                 |
| `ROWS_AFFECTED` | 影响的行数                 |
| `DIGEST`        | 语句指纹，用于归类相似语句 |

