## 动态映射

es默认会给传入index的数据的所有列创建索引，生产环境中，这是个危险的问题，长久使用可能会导致内存爆炸

### 配置动态映射

```
PUT /my_index
{
  ...
  "mappings": {
  	"dynamic": "strict",
    "properties": {
      "user_id": {
        "type": "keyword" 
      },
      "username": {
        "type": "text",
        "analyzer": "standard"
      },
      "age": {
        "type": "integer"
      },
      "created_at": {
        "type": "date",
        "format": "yyyy-MM-dd HH:mm:ss||yyyy-MM-dd||epoch_millis"
      }
    }
  }
}
```

* dynamic选择：

  * true：默认，给所有加入的数据的新字段添加索引

  * false：不给新加的数据的字段添加索引

  * strict：严格判断，如果写入的文档中包含了 Mapping 里没定义的字段，ES 直接抛出异常并**拒绝整条写入请求**。

    * 设定为strict后，后续可以通过更新mapping来增加允许的字段，更新语句参考如下

      ```
      curl --location --request PUT 'localhost:9200/user_index/_mapping' \
      --header 'Content-Type: application/json' \
      --data '{
        "properties": {
          "age": {
            "type": "integer"
          }
        }
      }'
      ```

    * 设定为strict后，无法删除已新增的mapping字段，只能新建索引转移

* dynamic类型和mapping字段的关系

  他们互相独立，dynamic的类型可以在index创建后随意修改，mapping中内容不会随着dynamic的修改而改变，只会因为dynamic对应类型的行为而改变。mapping中的内容在dynamic为任何类型时都生效，es都会给这里面对应的字段创建字段的索引。

  * 改变 `dynamic` 设置只会影响**后续**进入索引的文档。对于已经存储在磁盘上的旧数据，Elasticsearch 不会因为你修改了配置就自动回头去扫描并补充字段的索引。
