## 介绍

InnoDB 的 MVCC（多版本并发控制，**Multi-Version Concurrency Control**）是 MySQL 实现事务隔离和高并发性能的核心机制之一。MVCC 是 InnoDB 在同一行数据上维护多个“版本”，让读写操作尽可能不加锁，实现高并发。

## 本质：**隐藏字段 + undo 日志**

InnoDB 表中每行数据实际上都会隐式增加两个字段：

| 字段           | 作用                                         |
| -------------- | -------------------------------------------- |
| `trx_id`       | 插入/更新该行的事务ID（即当前事务的唯一ID）  |
| `roll_pointer` | 指向 undo log 的指针，可用于回溯旧版本的数据 |

还有一套 undo log 机制，**记录每次修改前的旧值（原始版本）**。这个 log 存在于 `ibdata` 文件中或 undo 表空间中。

* 每次事务中寻找数据时，会从 roll_pointer 串中寻找正确版本的数据。
